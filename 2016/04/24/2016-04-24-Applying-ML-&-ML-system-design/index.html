Applying ML & ML system design

发表于 2016-04-24   |   分类于 Machine Learning Course notes

Applying ML

Evaluate hypothesis

Split the dataset into training set and test set (often 70% and 30% respectively)

Learning the parameter θ using training set

Compute the test error θ using training set

Model Selection (choosing degree of polonomial or some constant like λ)

Split the dataset into training set CV set and test set (often 60%, 20% and 20% respectively)

Learning the parameter θ using training set under different λ

Compute the best constant (ie. λ) by computing CV set error

Compute the test error (cost function) using the test set under best constant (ie. λ)

High bias (underfit) VS High variance (overfit)

High Bias High Variance
Lagre λ Small λ

Learning Curves

Draw a curve (axis y = error, axis x = dataset size) to analyse whether the algorithm perform well

Debugging a learning algorithm

Solutions Effects
Get more training examples Fixed high variance
Try smaller sets of features Fixed high variance
Try getting additional features Fixed high bias
Try adding polynomial features Fixed high bias
Try decreasing λ Fixed high bias
Try increasing lambda Fixed high variance

ML System Design

Error analysis

  1. Start with a simple algorithm you can quickly implement. Implement it and test it on your CV data.
  2. Plot learning curves to decide if more data, more features, etc. are likely to help
  3. Error analysis: Manually examine the examples (in CV set) that your algorithm made errors on. See if you spot any systematic trend in what type of examples is making errors on.

    Error metrics for skewed classes

    | | | Actual|Class |

    | | | 1 | 0 |
    | Predicted |1 | True Positive| False Positive|
    | Class| 0 |False Negative | True Negative|

Data for ML

If the parameters are sufficient (a human expert can make correct prediction using the parameters) , then collecting more data is usually helpful
